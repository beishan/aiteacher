## 1. 项目概述

### 1.1 项目背景

随着教育需求的多样化和家教市场的扩大，家教老师在日常教学管理中面临大量繁琐的行政事务，包括学生信息管理、课程安排、作业批改、资料整理、学习跟踪等，严重占用了教学时间和精力。

家教助手系统旨在通过数字化手段帮助家教老师高效管理教学事务，并借助 AI 技术提升教学质量，让老师能够将更多精力专注于教学本身。

### 1.2 项目目标

- 构建一套完整的家教管理数字化平台，覆盖学生、排课、作业、资料全流程管理
- 通过 AI 辅助功能，提升作业批改、教学计划制定、学情分析的效率
- 支持多终端（PC、手机）访问，并与日历、通讯工具深度集成
- 系统部署于家庭 NAS 环境，保护数据隐私，支持局域网离线使用
- 提供开放的 AI 模型接入能力，支持主流大模型灵活切换

### 1.3 用户角色

| 角色 | 描述 | 主要使用场景 |
|------|------|-------------|
| 主教师（管理员） | 系统核心使用者，即家教老师本人 | 学生管理、排课、作业批改、AI辅助教学、数据分析、AI模型配置、系统参数设置、数据备份 |

## 2. 技术架构与开发规范

### 2.1 技术选型

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | Vue 3 + Vite | SPA 单页应用，支持响应式布局 |
| 后端 | Java 17 + Spring Boot 3.x | RESTful API，JWT 鉴权 |
| 数据库 | PostgreSQL 15+ | 关系型数据库，支持 JSON 字段扩展 |
| 缓存 | Redis | 会话管理、接口缓存、任务队列 |
| 文件存储 | MinIO / 本地挂载卷 | 资料文件存储，部署于 NAS |
| 容器化 | Docker + Docker Compose | 全服务容器化部署 |
| CI/CD | GitHub + Jenkins | 自动化构建与部署到 NAS |
| 反向代理 | Nginx | 静态资源、API 转发、HTTPS |
| AI 集成 | SpringAI  | 支持 Claude、ChatGPT、Minimax 等 |
| 日历集成 | CalDAV 协议 | 兼容 iOS / 小米日历同步 |
| 通知推送 | 企业微信 Webhook / APNs | 课程提醒推送 |

### 2.2 开发环境规范

- **开发机**：macOS，使用 Docker Desktop 运行本地服务
- **代码规范**：后端遵循 Google Java Style，前端遵循 Vue3 Composition API 风格
- **API 规范**：RESTful，版本前缀 `/api/v1`，统一响应格式
- **数据库**：使用 Flyway 进行数据库版本管理
- **Git 规范**：`feature/xxx` 分支开发，`main` 分支保护，PR 合并须通过 CI
- **资源文件**：放置于项目根目录 `resources/` 下，超过 10MB 的文件加入 `.gitignore`

### 2.3 生产部署架构

生产环境运行于家庭局域网内的飞牛 NAS Docker 服务中，部署流程如下：

1. 开发者推送代码至 GitHub 仓库
2. Jenkins 监听 GitHub Webhook，触发自动化构建
3. Jenkins 执行 Maven 构建（后端）+ `npm build`（前端）
4. 构建产物打包为 Docker 镜像，推送至本地私有 Registry
5. Jenkins 通过 SSH 远程执行 `docker-compose up -d` 完成部署
6. Nginx 反向代理对外提供服务，内网通过固定 IP 或域名访问

## 3. 功能模块详细需求

### 3.1 学生管理模块

#### 3.1.1 学生基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | 学生真实姓名 |
| 性别 | 枚举 | 是 | 男 / 女 / 其他 |
| 出生日期 | 日期 | 否 | 用于自动计算年龄 |
| 年龄 | 数字 | 否 | 可手动填写或由出生日期计算 |
| 年级 | 枚举 | 是 | 小学1-6年级、初一至初三、高一至高三 |
| 就读学校 | 文本 | 否 | 学生所在学校名称 |
| 学习科目 | 多选 | 是 | 语数英物化生历政地等多科目支持 |
| 学生来源 | 枚举 | 否 | 家长推荐 / 群推荐 / 平台引流 / 其他 |
| 家庭住址 | 文本 | 否 | 详细地址，用于判断上课地点 |
| 学生联系方式 | 文本 | 否 | 电话 / 微信 |
| 家长姓名 | 文本 | 否 | 主要联系人姓名 |
| 家长联系方式 | 文本 | 是 | 电话 / 微信，用于课程通知 |
| 家长关系 | 枚举 | 否 | 父亲 / 母亲 / 其他监护人 |
| 备注 | 长文本 | 否 | 特殊情况、注意事项等 |
| 入学日期 | 日期 | 是 | 开始辅导的日期 |
| 状态 | 枚举 | 是 | 在读 / 暂停 / 毕业 / 流失 |
| 学生头像 | 图片 | 否 | 可上传头像 |
| 标签 | 多选标签 | 否 | 自定义标签，如：基础薄弱、学习积极等 |

#### 3.1.2 课时费管理

支持灵活的计费模式，满足不同学生的缴费习惯：

- **按次计费**：每节课上完结算一次费用，系统自动记录每次上课及对应费用
- **按次预缴（包课）**：提前缴纳 N 次课费用，系统记录剩余课时数，课时不足时自动提醒
- **按时间段缴费**：缴纳某时间段（如某月、某学期）的固定费用，系统跟踪时间段内的课时消耗
- **自定义费率**：每个学生可设置不同课时单价，支持区分科目定价
- **费用记录**：记录每笔收款时间、金额、备注，支持导出账单
- **欠费提醒**：课时不足或到期前自动发送提醒通知
- **费用统计**：按月 / 学期统计收入，可视化图表展示

#### 3.1.3 学生状态追踪

- **学习进度**：记录每阶段学习目标及完成情况
- **知识点掌握度**：通过测试成绩和 AI 分析，生成知识点掌握热力图
- **出勤记录**：记录每次课程出勤 / 缺勤 / 请假情况
- **学情摘要**：AI 生成的定期学情摘要，可发送给家长

### 3.2 排课管理模块

#### 3.2.1 排课方式

1. **单次排课**：指定具体某天的某个时间段安排课程
2. **周期排课（每周）**：如每周日 10:00-12:00 固定上课，直到指定截止日期
3. **隔周排课**：如每两周上一节课，支持自定义间隔周数
4. **自定义周期**：支持按月、隔 N 天等自定义排课周期
5. **批量排课**：支持一次性为某学生排定整个学期的课程

#### 3.2.2 排课视图

- **日历视图**：提供月视图、周视图、日视图三种模式，支持自由切换
- **课程色彩标记**：不同学生 / 科目用不同颜色区分
- **拖拽调课**：在日历上直接拖拽修改课程时间
- **课程详情弹窗**：点击课程显示学生信息、科目、备注、费用等
- **冲突检测**：排课时自动检测时间冲突并给出提示
- **筛选过滤**：按学生、科目、课程类型筛选日历显示

#### 3.2.3 日历同步

支持将课程同步至手机日历，方便老师在手机端查看课程安排：

- **iOS 日历**：通过 CalDAV 协议，iOS 系统日历 App 订阅同步
- **小米日历**：通过 CalDAV 协议，小米手机日历 App 订阅同步
- **事件格式**：同步内容包含学生姓名、科目、上课地点、备注
- **双向同步**：支持在手机日历中修改后同步回系统（可选功能）
- **同步配置**：可自定义同步哪些课程类型，设置提醒时间

#### 3.2.4 课程类型

| 类型 | 说明 | 特殊配置 |
|------|------|---------|
| 单独授课（一对一） | 老师与单个学生的私人辅导课 | 仅关联一名学生 |
| 网课 | 线上授课，通过视频会议工具进行 | 可记录会议链接 / 平台信息 |
| 班科（小班课） | 多名学生组成小班一起上课 | 关联虚拟班级，共享课程资料 |

#### 3.2.5 课程提醒

- **企业微信推送**：通过企业微信机器人 Webhook，在上课前 N 分钟推送提醒消息
- **iCal 提醒**：日历同步事件自带提醒，手机系统提醒事项自动弹出
- **提醒内容**：学生姓名、课程时间、科目、地点、本次课程备注
- **家长通知**：上课前可自动向家长发送上课提醒（可选开关）
- **课后总结推送**：课程结束后，可自动推送课程总结至家长（可选）

### 3.3 作业管理模块

#### 3.3.1 作业基本功能

- **创建作业**：支持为单个学生或班级创建作业，设置截止日期
- **作业内容**：支持文字描述、图片、PDF 附件等多种形式
- **作业提交**：支持上传作业照片（手写）、文档、图片等
- **作业状态**：未提交 / 已提交 / 已批改 / 已查看
- **打分系统**：支持百分制、五星制、等级制（优 / 良 / 中 / 差）等多种评分方式
- **评语功能**：支持文字评语，支持语音转文字评语（快速录入）
- **作业历史**：按学生查看历史所有作业记录，追踪成长轨迹

#### 3.3.2 AI 批改作业

- **多图上传**：支持一次上传多页作业照片
- **OCR 识别**：调用视觉 AI 模型识别手写内容
- **逐题批改**：AI 按题目编号逐一批改，标记对错
- **详解生成**：对错误题目生成详细的解题步骤
- **批改报告**：生成结构化批改报告，含总分、错题列表、知识点分析
- **人工复核**：老师可对 AI 批改结果进行修改和补充
- **历史对比**：与该学生历次同类作业对比，分析进步情况

#### 3.3.3 作业分析统计

- **班级整体分析**：同一作业不同学生的完成情况对比
- **个人错误趋势**：某学生历次作业的错误知识点追踪
- **完成率统计**：按时间维度统计学生作业完成率
- **作业量统计**：按周 / 月统计布置的作业量，避免超载

### 3.4 虚拟班级管理模块

- **班级创建**：为需要一起上课的多名学生建立虚拟班级
- **成员管理**：添加 / 移除班级成员，设置班级状态（开课中 / 结束）
- **统一排课**：为整个班级排课，自动关联所有成员
- **统一作业**：为整个班级布置作业，每个学生独立提交和批改
- **课程讲义**：班级共享讲义资料，统一上传和查看
- **班级记录**：记录每次班课的内容、进度、出勤情况
- **费用分摊**：班科费用可按学生数量自动分摊计算

### 3.6 资料库管理模块

#### 3.6.1 资料库基本功能

- **资料上传**：支持 Word、PDF、Excel、常见图片格式（JPG / PNG / GIF / WEBP）
- **目录管理**：支持文件夹层级分类，自定义目录结构
- **资料搜索**：全文搜索资料内容（PDF / Word），以及按标签、科目、年级搜索
- **收藏功能**：标记常用资料，快速访问收藏清单
- **资料标签**：为资料打上知识点、年级、科目等多维度标签
- **分享链接**：生成资料的临时访问链接，可分享给学生 / 家长（限时有效）
- **资料清单**：导出选定资料的清单（含文件名、大小、修改日期）

#### 3.6.2 学生专属资料库

- **专属文件夹**：为每个学生建立独立的资料空间
- **资料分配**：将公共资料库的资料分配到学生专属库
- **班级资料共享**：班级成员共享一个资料空间
- **访问权限**：控制每个资料对哪些学生可见

#### 3.6.3 资料多版本管理

- **版本保存**：每次修改资料时，旧版本自动保存
- **版本历史**：查看某资料的所有历史版本，含修改时间和修改人
- **版本对比**：对比不同版本之间的差异
- **版本恢复**：可将资料恢复到任意历史版本
- **版本清理**：支持手动清理不需要的历史版本以节省空间

#### 3.6.4 资料预览与编辑

| 格式 | 预览 | 在线编辑 | 说明 |
|------|------|---------|------|
| PDF | 支持 | 批注 / 填写 | 通过 PDF.js 预览，支持高亮批注 |
| Word (.docx) | 支持 | 支持 | 基于 LibreOffice Online 或 ONLYOFFICE |
| Excel (.xlsx) | 支持 | 支持 | 表格在线查看与编辑 |
| 图片 (JPG/PNG等) | 支持 | 裁剪 / 标注 | 支持旋转、标注、画圈等简单编辑 |
| 图片 (GIF/WEBP) | 支持 | 不支持 | 仅预览 |

## 4. AI辅助教学功能

### 4.1 AI 模型接入架构

系统采用统一的 AI 网关层，对接多个主流大模型，老师可在设置中选择偏好模型：

| 模型 | 提供商 | API类型 | 适用场景 |
|------|--------|---------|---------|
| Claude 3.5 / Claude 4 | Anthropic | REST API | 作业批改、文字分析、教学计划 |
| GPT-4o / GPT-4 | OpenAI | REST API | 通用问答、内容生成 |
| Minimax MoE | MiniMax | REST API | 中文教学场景优化 |
| Xiaomi MiMo | 小米 | REST API | 本地化中文理解 |
| 通义千问 Qwen | 阿里云 | REST API | 中文教育内容生成 |
| 自定义模型 | 用户自定义 | OpenAI 兼容格式 | 支持本地部署模型（如 Ollama）|

AI 网关层统一处理：提示词构建、模型切换、异常重试、响应解析、Token 消耗统计。

### 4.2 AI 批改作业

- **多图上传**：支持一次上传多页作业照片
- **OCR 识别**：调用视觉 AI 模型识别手写内容
- **逐题批改**：AI 按题目编号逐一批改，标记对错
- **详解生成**：对错误题目生成详细的解题步骤
- **批改报告**：生成结构化批改报告，含总分、错题列表、知识点分析
- **人工复核**：老师可对 AI 批改结果进行修改和补充
- **历史对比**：与该学生历次同类作业对比，分析进步情况
- **作业难度评估**：AI 评估作业难度是否与学生水平匹配
- **改进建议**：针对高频错误给出后续教学建议

### 4.3 AI 教学计划制定

- **基于学情**：结合学生成绩、作业情况、知识点掌握度生成个性化计划
- **目标设定**：老师输入目标（如：期末考试提升20分），AI 制定阶段计划
- **课程讲义生成**：AI 根据教学计划自动生成对应课程讲义（Word 格式）
- **作业题目生成**：AI 生成与讲义配套的练习题
- **计划调整**：根据实际学情自动调整后续计划
- **里程碑设定**：设定关键时间节点目标，追踪达成情况

### 4.4 AI 学生资料库与分析（AI Student Profile）

AI 学生资料库是每个学生的智能数字档案，持续积累和分析学生学情：

- **基础档案**：汇总学生基本信息、学习背景、个性特点
- **知识图谱**：建立学生专属的知识点掌握度图谱，可视化呈现薄弱点
- **学习风格分析**：通过作业完成情况、错误类型分析学生学习风格
- **历史记录时间线**：完整记录每次课程内容、作业成绩、阶段测试
- **情绪与状态追踪**：记录学生课堂状态（积极 / 消极 / 注意力不集中）
- **教学建议引擎**：在老师查看学生档案时，AI 实时生成教学建议
- **家长汇报生成**：一键生成适合向家长汇报的学情总结报告
- **风险预警**：当学生成绩明显下滑或长期缺课时，自动发出预警

### 4.5 AI 智能问答助手

集成对话式 AI 助手，老师可用自然语言查询系统信息：

- **排课查询**："下周有哪些课？" "小明这周几点上课？"
- **学情查询**："张同学最近学的什么内容？" "谁的数学成绩最差？"
- **作业查询**："还有哪些作业没批改？" "李华最近作业完成情况怎么样？"
- **费用查询**："本月收了多少学费？" "哪个学生课时快用完了？"
- **教学建议**："小红下次课应该讲什么？" "给王峰出5道二次方程题"
- **报告生成**："帮我生成张三本月的学情报告"
- **上下文理解**：支持多轮对话，记住对话上下文

## 5. 系统功能与非功能需求

### 5.1 用户认证与权限

- **本地账号体系**：用户名+密码登录，JWT Token 鉴权
- **双因素认证（可选）**：支持 TOTP 动态码二次验证
- **会话管理**：支持多设备登录，可查看并踢出其他登录会话
- **权限控制**：基于角色的权限管理（RBAC），预留学生 / 家长角色扩展
- **操作日志**：记录关键操作，可追溯

### 5.2 通知与消息系统

- **企业微信**：配置 Webhook，发送课程提醒、作业提醒、欠费提醒
- **系统内通知**：站内通知中心，集中显示所有提醒事项
- **邮件通知（可选）**：支持通过 SMTP 发送邮件通知
- **提醒配置**：可自定义各类提醒的触发条件和时间

### 5.3 数据统计与报表

- **收入统计**：按日 / 周 / 月 / 学期统计收入，展示趋势图
- **课时统计**：统计每个学生的上课总时长
- **学生分布**：按年级、科目、来源等维度统计学生分布
- **作业统计**：作业完成率、平均分等汇总统计
- **AI 使用统计**：各 AI 模型使用次数、Token 消耗量统计
- **报表导出**：支持导出 Excel 格式的统计报表

### 5.4 数据管理

- **数据备份**：自动定时备份 PostgreSQL 数据库到 NAS 指定目录
- **数据导入**：支持从 Excel 导入学生基本信息、历史成绩
- **数据导出**：支持导出学生档案、费用记录、成绩数据
- **数据删除**：支持软删除（回收站机制），防止误删

### 5.5 系统设置

- **AI 模型配置**：在设置页配置各 AI 模型的 API Key，选择默认模型
- **通知配置**：配置企业微信 Webhook、提醒时间等
- **日历同步配置**：CalDAV 服务器地址、账号设置
- **费用设置**：设置默认课时费，配置货币单位
- **科目管理**：自定义教授科目列表
- **标签管理**：自定义学生标签、资料标签

### 5.6 非功能需求

| 类别 | 需求 | 指标 |
|------|------|------|
| 性能 | 页面加载速度 | 首屏加载 < 3秒（局域网内）|
| 性能 | API 响应时间 | 普通接口 < 200ms，AI 接口依赖模型响应 |
| 可用性 | 系统可用率 | 家庭 NAS 环境，支持优雅停机 |
| 安全性 | 数据传输 | 内网 HTTPS，JWT Token 有效期可配置 |
| 安全性 | 文件访问 | 资料文件需经过鉴权，禁止直接 URL 访问 |
| 兼容性 | 浏览器支持 | Chrome / Safari / Edge 最新版本 |
| 兼容性 | 移动端 | 响应式设计，支持手机浏览器访问 |
| 可维护性 | 日志 | 结构化日志，保留最近 30 天 |
| 存储 | 文件存储 | 支持 NAS 扩展存储，单文件不超过 500MB |
| 扩展性 | AI 模型 | 插件化设计，新增模型只需实现统一接口 |

## 8. 附录

### 8.1 术语表

| 术语 | 解释 |
|------|------|
| CalDAV | 日历同步协议，用于在服务器和客户端间同步日历数据（iOS / 安卓均支持）|
| RBAC | 基于角色的访问控制（Role-Based Access Control）|
| JWT | JSON Web Token，用于无状态的 API 鉴权 |
| Webhook | HTTP 回调，企业微信机器人通过 Webhook 接收消息 |
| OCR | 光学字符识别，用于从作业图片中识别文字 |
| CalDAV Server | CalDAV 协议服务端，由本系统后端实现，供手机日历客户端连接 |
| Token 消耗 | 调用 AI API 时消耗的计算单位，影响 API 费用 |
| 虚拟班级 | 系统内将多名学生组成的逻辑班级，不对应真实学校班级 |
